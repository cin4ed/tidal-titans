## Project overview

A browser demo of a **Wind WakerвЂ“style stylized ocean** with a navigable procedural sailboat.

- **Language/UI**: Spanish copy in the HTML; code comments in English.
- **Entry point**: `src/main.js` (single file, ~390 lines).
- **Framework**: [Vite](https://vitejs.dev/) `^6.2.0` + [Three.js](https://threejs.org/) `^0.160.0` (ES modules).
- No TypeScript, no component framework, no state manager.

## src/main.js architecture

Everything lives in one file. The sections in order:

### 1. Wave math (CPU side)

```
waveHeight(x, z, time)  вЂ” layered sine waves, returns Y displacement
waveNormal(x, z, time)  вЂ” finite-difference gradient of waveHeight
```

**Critical**: the exact same formula must exist in the GLSL vertex shader. If you change the wave parameters in JS, update the shader too (and vice versa).

### 2. Scene setup

- `THREE.Scene` with `THREE.Fog(0x87ceeb, 35, 120)`
- `PerspectiveCamera` (FOV 55, near 0.1, far 500)
- `WebGLRenderer` (antialias, sRGB output, `pixelRatio` capped at 2)
- Sky: 2Г—256 canvas gradient texture used as `scene.background`
- Sun: `DirectionalLight` + decorative `SphereGeometry` positioned along `sunDir`

### 3. Ocean ShaderMaterial

- Geometry: `PlaneGeometry(220, 220, 160, 160)` rotated вЂ“90В° on X
- **Vertex shader**: displaces vertices in world space using `waveHeight`, computes per-vertex normal via finite differences
- **Fragment shader**: toon color bands from wave height, cel-stepped NВ·L lighting, Fresnel edge highlight, foam at crests, manual distance fog matching `scene.fog`
- Uniforms updated every frame: `uTime`, `uCameraPos`, (fog is baked into uniforms not `THREE.Fog`)

### 4. Boat (procedural)

- `THREE.Group` containing flat-shaded `MeshStandardMaterial` meshes:
  - Hull: `BoxGeometry`
  - Bow: `ConeGeometry` (rotated to point forward)
  - Deck: `BoxGeometry` (lighter wood)
  - Mast: `CylinderGeometry`
  - Sail: `PlaneGeometry` with per-vertex bulge, `DoubleSide`
  - Flag: `PlaneGeometry`, `MeshBasicMaterial` red
- Group `rotation.y = Math.PI` so the bow faces the movement direction (+Z in world)

### 5. Input

- WASD keys tracked in a `keys` object via `keydown`/`keyup`
- Pointer Lock: click canvas в†’ `requestPointerLock()`, `pointerlockchange` updates `pointerLocked`, `mousemove` applies `movementX/Y` to `camOrbit.yaw/pitch`
- Hint text updates dynamically when lock state changes

### 6. Orbit camera state

```js
camOrbit = { yaw, pitch, distance: 12, sensitivity: 0.0022, pitchMin, pitchMax }
```

- `yaw` is **absolute world angle** (not relative to boat) so the camera doesn't spin when the boat turns
- Mouse up в†’ `pitch += movementY * sensitivity` в†’ pitch decreases в†’ camera lowers в†’ more sky visible (natural feel)
- Camera position derived from spherical coordinates; `lerp` at 0.1 for smoothness; `lookAt` the point 1.5u above boat

### 7. Animation loop

`renderer.setAnimationLoop(...)` runs each frame:

1. Update ocean uniforms
2. Accelerate/brake/drag `speed`; rotate `boat.rotation.y` on A/D
3. Move boat along `boatForward` vector
4. Sample `waveHeight` for boat Y; `waveNormal` for pitch/roll lean
5. Place camera on orbit sphere around boat anchor; `lookAt`

## Key constraints and gotchas

1. **Wave math must stay in sync** вЂ” JS `waveHeight` and the GLSL `waveHeight` function use identical parameters. Diverging them breaks the boat bobbing.
2. **Camera yaw is world-absolute** вЂ” do not make yaw relative to `boat.rotation.y` or the camera will spin when the boat turns.
3. **Pitch sign** вЂ” `camOrbit.pitch += e.movementY * sensitivity` (positive, not negative). Inverting breaks natural look-up/down feel.
4. **Ocean fog is manual** вЂ” `scene.fog` is set for other objects, but the ShaderMaterial handles its own fog via `uFogColor/Near/Far` uniforms. Keep both consistent.
5. **No per-frame allocation** вЂ” vectors (`boatForward`, `camTarget`, `camDesired`, `rightVec`, `worldUp`) are declared once outside the loop and reused.
6. **Single-file convention** вЂ” the user prefers all application logic in `src/main.js`. Do not split into multiple modules without being asked.
7. `**dist/` may be committed** вЂ” the `.gitignore` lists `dist` but some build artifacts appear tracked. Don't delete `dist/` without checking `git status`.
