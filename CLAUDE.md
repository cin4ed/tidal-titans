# CLAUDE.md — tidal-titans

This file tells Claude how this repository is structured, what it contains, and how to work in it effectively.

---

## Project overview

A browser demo of a **Wind Waker–style stylized ocean** with a navigable **procedural pirate ship**, **port and starboard broadsides** (charge + trajectory preview), and a **static enemy sloop** with a **world-space health bar** (visual only for now).

- **Language/UI**: English copy in the HTML; code comments in English.
- **Entry point**: `src/main.js` — async `init()` (~840 lines) that sets up WebGPU, scene, TSL water, post-processing, player and enemy ships, input, cannons, and the animation loop.
- **Stack**: [Vite](https://vitejs.dev/) `^6.2.0` + [Three.js](https://threejs.org/) `^0.183.0` (ES modules).
- **Analytics**: [`@vercel/analytics`](https://vercel.com/docs/analytics) — `inject()` runs once at module load in `main.js` (production-friendly; no extra setup in this repo).
- **Rendering**: **WebGPU** via `three/webgpu` — not `WebGLRenderer`. Node materials and **TSL** (`three/tsl`) drive the water surface, sky background, and fullscreen post output.
- **Dev tooling**: [Tweakpane](https://tweakpane.github.io/docs/) v4 (devDependency) — live tuning; loaded only when `import.meta.env.DEV` is true.
- No TypeScript, no component framework, no state manager.

**Browser note**: Users need WebGPU support (current Chrome/Edge, Safari Technology Preview, etc.). There is no WebGL fallback in this repo.

## src/config.js

Default export: **`config`**. The dev panel edits this object in memory; **Export Config** copies a full `config.js` snippet to the clipboard — paste into `src/config.js` to persist (see `serializeConfig()` in `dev-panel.js` for the exact emitted shape).


| Group                 | Path                                                                                | Role                                                                                                                                                                                                                                                                                                                                                                                    |
| --------------------- | ----------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Wave layers           | `config.waves.wave1`–`wave4` — each `{ freq, speed, amp }`                          | Used by **CPU** `waveHeight` / `waveNormal` (boat and enemy bob/tilt) and mirrored in **TSL** vertex displacement (scaled by `config.water.waveVisualScale` for the mesh).                                                                                                                                                                                                              |
| Wave spatial scale    | `config.waves.spatialScale`                                                         | **Proportional world scaling** of the sine wave field: divides X/Z inside the wave phase (longer wavelengths, fewer crests per area when > 1) and multiplies each layer’s **amp** by the same factor so steepness stays consistent. Synced to TSL; affects **both** physics and GPU displacement (unlike `waveVisualScale`). Dev panel: **Spatial scale** under Boat Wave Physics.       |
| Wave amplitude scale  | `config.waves.ampScale`                                                             | Global multiplier for **all four** layer **amp** values at once (after spatial scaling in the height formula). Synced to TSL; affects **both** physics and GPU displacement. Dev panel: **All layers amp** under Boat Wave Physics.                                                                                                                                                  |
| Water colors          | `config.water.colorDeep`, `config.water.colorLight` (hex)                           | Worley-mixed surface tint in TSL; driven by uniform colors synced each frame from `config`. `colorDeep` is also the scene **fog** tint when fog is enabled.                                                                                                                                                                                                                             |

Independent from **spatialScale**.                                                                                                                                                                                                                                                           |

| Refraction            | `config.water.refractionStrength`                                                   | Screen-space UV offset strength.                                                                                                                                                                                                                                                                                                                                                        |

| Fog                   | `config.fog.enabled`, `config.fog.near`, `config.fog.far`                           | When enabled, `THREE.Fog` on the scene (boat, underwater props, sun, etc.); near/far/color synced from config each frame.                                                                                                                                                                                                                                                              |
| Orbit zoom            | `config.camera` — `distanceInitial`, `distanceMin`, `distanceMax`, `distanceStep`    | Scroll-wheel stepped dolly on `camOrbit.distance`; initial value clamped to min/max.                                                                                                                                                                                                                                                                                                    |

## src/main.js architecture

`main.js` imports `three/webgpu` and `three/tsl`. Initialization is **async** because **`await renderer.init()`** is required for WebGPU.

### 1. Wave math (CPU — boat + enemy physics)

- **`waveHeight(x, z, t)`** — layered sine recipe **mirrored in TSL** for the water mesh (must stay aligned). Uses `config.waves.spatialScale` and `config.waves.ampScale` (clamped).
- **`waveNormal(x, z, t)`** — finite-difference gradient for pitch/roll.

These read `config.waves.wave1`–`wave4`, `spatialScale`, and `ampScale`. They do **not** use `config.water.waveVisualScale`; that affects only the rendered water mesh height multiplier.

*(An inline comment near `waveHeight` mentions dropping a “GLSL/TSL sync constraint”; in practice **CPU and TSL wave formulas still must match** — see TSL block comments that say they mirror `waveHeight()` exactly.)*

### 2. Scene & renderer

- **`THREE.Scene`** + optional **`THREE.Fog`** when `config.fog.enabled`: color from `config.water.colorDeep`, distances from `config.fog.near` / `far` (synced each frame when on).
- **`scene.backgroundNode`**: TSL mix on `normalWorld.y` between `config.sky.horizon` and `config.sky.zenith` — **not** tied to water colors.
- **`THREE.PerspectiveCamera`** (FOV 55, **near 0.25**, far 500).
- **`THREE.WebGPURenderer`**: antialias, sRGB output, pixel ratio capped at 2.

### 3. Lighting & content

- **Sun**: `DirectionalLight` along `sunDir` + decorative `MeshBasicMaterial` sphere.
- **Fill**: two `HemisphereLight`s for water/sky ambience.
- **Underwater props**: loop of `IcosahedronGeometry` + `MeshStandardMaterial`, subtle animation in the loop (refraction/depth showcase).

### 4. TSL water (`MeshBasicNodeMaterial`)

- **Geometry**: `PlaneGeometry(220, 220, 200, 200)` rotated −90° on X (lies in XZ).
- **Vertex**: `positionLocal` displaced in Y by TSL that **mirrors** `waveHeight`, then multiplied by `uniform(config.water.waveVisualScale)`.
- **Uniforms**: wave layers, Worley/refraction, water deep/light, sky horizon/zenith — synced each frame from `config`.
- **Fragment**: Worley-based `waterColor`; **backdrop** samples the scene via `viewportSharedTexture` with refraction UVs and depth tests. Transparency enabled.

### 5. Ships

- **`createPirateShip()`** from `./models/pirateShip.js` returns `{ group, mainSail, topSail, foreSail, flagMain, portCannonMuzzleLocal, starboardCannonMuzzleLocal, mainSailBaseZ, topSailBaseZ, foreSailBaseZ }`.
- **`portCannonMuzzleLocal` / `starboardCannonMuzzleLocal`**: three `THREE.Vector3` offsets each (ship-local) for broadside muzzles; used for spawning shots and trajectory preview.
- The **`group`** uses **`rotation.y = Math.PI`** so local +Z aligns with navigation/rigging.
- **`createEnemyShip()`** — procedural enemy sloop; spawned at a fixed offset, bobbing on waves like the player. **`createEnemyHealthBar()`** — sprite with `setHealth()`; position updated from a mast-local offset each frame. **`enemyHealth` is currently held at 1** — there is **no** projectile–hull hit test yet (placeholder for combat feedback).

### 6. Input

- **WASD** in a `keys` map (`keydown` / `keyup`).
- **Pointer lock** on canvas click; `mousemove` updates **`camOrbit.yaw`** / **`pitch`** when locked.
- **Scroll wheel** on the canvas (non-passive): stepped zoom on **`camOrbit.distance`** using **`config.camera`**; distance is clamped each frame for live dev-panel edits.
- **`#hint`** in **`index.html`**: bottom-left panel with illustrated rows; **`main.js`** toggles **`is-locked`** on pointer lock. Cannon row documents **Q / E** broadsides (not mouse buttons).

### 7. Broadsides (combat)

- **When**: Only while **pointer is locked** on the canvas (`document.pointerLockElement === renderer.domElement`).
- **Charge / fire**: **`keydown`/`keyup` on Q (port) and E (starboard)** — hold to charge, **release** to fire a **three-gun volley** with stagger from **`config.combat.volleyStagger`**. Hold duration maps to power between **`powerMin`** and **`powerMax`** over **`maxChargeTime`**. **Separate cooldown timers** for port and starboard (`cannonCooldownUntilPort` / `cannonCooldownUntilStarboard`).
- **Cancel charge**: Losing pointer lock clears charging **without** firing.
- **Ballistics**: World-space spheres parented to **`scene`**; initial velocity uses broadside horizontal direction (±`rightVec` in XZ), **`launchAngleDeg`** for elevation, **`muzzleSpeed * powerScale`**, plus **`boatForward * speed`**. Integration: **`velocity.y -= config.combat.gravity * dt`**. Per-shot **max horizontal range** and **max lifetime** scale with power (`rangeAtMinPower` / `rangeAtMaxPower`).
- **Trajectory preview**: While charging (and that side off cooldown), a **strip mesh** (indexed triangle ribbon) shows the arc — **additive, semi-transparent** `MeshBasicMaterial`, camera-independent width via **`trajectoryRibbonWidth`**. Preview muzzle index: **`trajectoryPreviewMuzzleIndex`** (port) and **`trajectoryPreviewStarboardMuzzleIndex`** (starboard, falls back to port index if unset). Integration uses **`trajectorySampleDt`** and up to **`trajectoryMaxSteps`** (clamped to an internal buffer cap). Stops when below **`waveHeight(x, z, t)`**, beyond preview range, or step cap.
- **`prepareCannonMuzzle`** nudges the spawn along the broadside by a fixed offset so balls clear the hull.

### 8. Orbit camera

```js
camOrbit = { yaw, pitch, distance, sensitivity: 0.0022, pitchMin, pitchMax }
// distance starts as clamp(config.camera.distanceInitial, min, max)
```

- **`yaw`** is **world-absolute** (not parented to boat yaw).
- **`pitch += movementY * sensitivity`** for natural look.
- Camera **lerps** toward a spherical offset from **`boat.position + (0, 1.5, 0)`** and **`lookAt`** that target.

### 9. Post-processing (`THREE.RenderPipeline`)

- **`pass(scene, camera)`** → color + linear depth.
- **Gaussian blur** on color; blur strength blended via a **mask** derived from scene depth / screen UV (see `waterMask` in code — not a literal water-line texture).
- **Output**: masked blend between blurred scene and a tinted + vignette path (`0x74ccf4` tint — align with palette when changing water colors).

Final draw: **`renderPipeline.render()`** (not `renderer.render(scene, camera)` alone).

### 10. Animation loop

1. **`timer.update()`** — `THREE.Timer` for stable `dt` / elapsed `t`.
2. Sync TSL uniforms from **`config.waves`**, **`config.water`**, **`config.sky`**; sync **`scene.fog`** when fog is enabled.
3. Underwater mesh motion.
4. Sail / flag flutter using **`boatObj.*`** mesh refs.
5. Boat movement from **`config.boat`**; **`waveHeight`** / **`waveNormal`** for player; enemy ship same wave sampling with its own offset.
6. Enemy health bar world position.
7. Cannon **shot queue** and **projectiles**; **trajectory ribbon** update while charging.
8. Camera lerp + **`lookAt`**.
9. **`renderPipeline.render()`**.

### 11. Dev panel (conditional)

```js
if (import.meta.env.DEV) {
  import('./dev-panel.js').then(({ mountDevPanel }) => {
    mountDevPanel(config, {
      onCameraInitialZoomChange: (d) => { /* sync camOrbit.distance */ },
    });
  });
}
```

- Folders include **Boat Wave Physics**, **Water Appearance**, **Sky**, **Depth Effect**, **Fog**, **Camera / Orbit**, **Boat Movement**, and **Combat / Cannons**. **Export Config** serializes waves through combat (including **`boat`**).

**`src/dev-panel.js`**: `mountDevPanel()` wires Tweakpane; `serializeConfig()` powers **Export Config** — when adding new **`config.*`** fields, extend **`serializeConfig()`** so pasted snippets stay complete. Tweakpane has no built-in per-control description API for nested controls; **Combat / Cannons** and **Boat Movement** use `bindWithHint` → `bindRowHoverHint` with **`#tp-dev-binding-hint`** on `document.body`. Hint copy should stay aligned with behavior in `main.js`.

Production builds omit this import via tree-shaking.

## Key constraints and gotchas

1. **Wave formula: JS ↔ TSL** — `waveHeight()` in JS and the TSL vertex displacement (same sines, same `config.waves.wave1`–`wave4` inputs) must stay identical or the boat (and enemy) will not sit on the waves you see. **`config.waves.spatialScale`** and **`config.waves.ampScale`** must match in both. **`config.water.waveVisualScale`** only scales the final GPU vertex offset, not CPU physics.
2. **`config.js` is the source of truth for tuning** — prefer exporting from Tweakpane into `config.js` rather than scattering magic numbers. Sky is **`config.sky.horizon` / `zenith`**. Scene fog color tracks **`config.water.colorDeep`**. **Boat** and **combat** are read live in the loop. Some aesthetic colors still live in `main.js` (e.g. post vignette tint `0x74ccf4`, lights, trajectory ribbon material); align with palette when retuning.
3. **No duplicate GLSL ocean shader** — everything water-related is TSL in `main.js`.
4. **Camera yaw is world-absolute** — do not derive yaw from `boat.rotation.y` or the camera will corkscrew when turning.
5. **Pitch sign** — keep `camOrbit.pitch += e.movementY * sensitivity` as-is for natural mouse-look.
6. **Scene fog vs water** — opaque/semi-opaque objects use `scene.fog`; the water material uses its own depth/backdrop logic.
7. **Reuse vectors in the loop** — `boatForward`, `camTarget`, `camDesired`, `rightVec`, `worldUp`, enemy helpers, etc., are allocated once outside `setAnimationLoop`.
8. **Tweakpane is dev-only** — do not import Tweakpane from `main.js` or `config.js` in a way that ships to production.
9. **`dist/` and `public/`** — `.gitignore` ignores `dist/` but your clone may still track builds; avoid deleting tracked artifacts without checking `git status`. **`public/`** may be missing; Vite still builds; add the folder when you need static URLs.
10. **WebGPU init** — any new setup that runs before the first frame must respect **`await renderer.init()`** and keep the async `init().catch(...)` entry pattern.
11. **Cannon trajectory buffer** — `trajectoryMaxSteps` in config is clamped to **`CANNON_TRAJ_BUFFER_STEPS`** (800) in `main.js` so preview buffers are not resized at runtime; raise that constant if you need longer preview ribbons.
12. **Vercel Analytics** — `inject()` is side-effectful at import time; keep it if you rely on page-view analytics on deployed builds.
